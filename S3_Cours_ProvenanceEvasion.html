1. Switch non manageable vs switch manageable

📍 Mise en situation : Vous avez maintenant un plan d'adressage complet pour les 4 sites de Provence Évasion. Avant de configurer quoi que ce soit, une question fondamentale se pose : avec quels équipements va-t-on bâtir ce réseau ? Tous les switches ne se valent pas. Certains font juste "passer" des trames, d'autres permettent une vraie segmentation, du contrôle, de la sécurité. C'est précisément ce que nous allons étudier aujourd'hui.

1.1 Le switch non manageable — "plug and play"

Un switch non manageable (aussi appelé switch non administrable) est le type de switch le plus simple. Il n'a pas d'interface de configuration : on le branche, il fonctionne immédiatement.

⚠ Switch non manageable
  • Aucune interface de configuration (pas d'IP de gestion)
  • Pas de notion de VLAN
  • Tous les ports sont dans le même domaine de broadcast
  • Pas de contrôle du trafic possible
  • Adapté pour : un bureau à domicile, un petit réseau sans segmentation
  • Prix : moins cher, mais sans souplesse
✅ Switch manageable (administrable)
  • Interface de configuration : CLI Cisco IOS (ou interface web)
  • Adresse IP de gestion (VLAN d'administration)
  • Création de VLANs, affectation des ports
  • Ports trunk pour interconnexion d'équipements
  • QoS, sécurité des ports, listes de contrôle d'accès
  • Adapté pour : tout réseau professionnel multi-VLANs
📌 À retenir — Pourquoi Provence Évasion a besoin de switches manageables

Sur chaque site de Provence Évasion, plusieurs types d'utilisateurs cohabitent sur le même câblage physique : le personnel, les clients WiFi, les caméras de surveillance. Si l'on utilisait des switches non manageables, tous ces équipements se retrouveraient dans le même réseau — ce qui poserait des problèmes graves :

  • Un client pourrait tenter d'accéder aux postes du personnel
  • Les caméras seraient visibles de tout le monde
  • Le trafic WiFi clients (très volumineux) ralentirait le réseau professionnel

La solution : des switches manageables Cisco 2960, configurés avec des VLANs distincts par type d'usage.

1.2 Rappel : qu'est-ce qu'un domaine de broadcast ?

Lorsqu'un équipement envoie une trame en broadcast (adresse de destination FF:FF:FF:FF:FF:FF), cette trame est reçue par tous les équipements du même réseau logique. On appelle cet ensemble un domaine de broadcast.

Sans VLAN, un switch non manageable à 24 ports forme un seul domaine de broadcast : une trame broadcast envoyée par un PC est reçue par les 23 autres ports. Cela génère du trafic inutile, et surtout, aucune isolation entre les équipements.

📌 À retenir — VLAN et domaine de broadcast

Chaque VLAN crée un domaine de broadcast distinct. Un switch manageable avec 3 VLANs, c'est comme si vous aviez 3 switches virtuels indépendants sur le même matériel physique. Les broadcasts restent confinés à leur VLAN : un broadcast du VLAN Personnel n'atteint jamais le VLAN Clients WiFi.

2. Les VLANs — Virtual Local Area Networks

2.1 Définition et principe

Un VLAN (Virtual Local Area Network) est un réseau logique créé par configuration logicielle sur un switch manageable. Il permet de regrouper des ports du switch dans un même réseau, indépendamment de la disposition physique des équipements.

Concrètement : sur un switch de 24 ports, on peut décider que les ports 1 à 8 appartiennent au VLAN 21 (Personnel), les ports 9 à 16 au VLAN 22 (Clients WiFi), et les ports 17 à 20 au VLAN 23 (Caméras). Ces trois groupes ne peuvent pas communiquer entre eux directement — ils sont logiquement isolés, même s'ils partagent le même matériel.

🔒 VLANs et sécurité — pourquoi c'est important pour Provence Évasion

La segmentation par VLAN répond directement aux exigences du cahier des charges d'Axentis :

  • Isolation des clients WiFi : un client de l'hôtel ne peut pas "voir" les postes du personnel ni accéder aux serveurs du siège.
  • Protection des caméras : les caméras IP ont leur propre VLAN (23, 33, 43…). Seul le NVR (enregistreur) du siège d'Apt peut y accéder. Cela évite qu'une personne malveillante récupère le flux vidéo.
  • Sécurité comptable : à Apt, le VLAN 11 (Personnel) isole les postes de comptabilité des autres équipements.
  • VLAN d'administration : le VLAN 14 regroupe les adresses IP de gestion des switches et routeurs. Seul le technicien peut y accéder.

2.2 Identifiant de VLAN (VID)

Chaque VLAN est identifié par un numéro appelé VID (VLAN Identifier), compris entre 1 et 4094. Par convention Cisco, le VLAN 1 est le VLAN par défaut (tous les ports y appartiennent à la sortie d'usine) — il est recommandé de ne pas l'utiliser pour les données.

Dans le projet Provence Évasion, les VIDs suivent la convention d'adressage déjà établie :

VIDNom du VLANUsageSites concernés
x1PersonnelPostes fixes, imprimantes réseauTous
x2Clients WiFiAccès Internet invités, isoléTous
x3Caméras IPSurveillance vidéo, NVRTous
x4AdministrationGestion des équipements réseauTous
x5ServeursDNS, NAS, NVR centraliséApt uniquement
x6SéminairesSalle séminaire extensibleSaignon uniquement

Rappel : x = identifiant du site (1=Apt, 2=Roussillon, 3=Saignon, 4=Cucuron). Exemple : VLAN 21 = Roussillon Personnel, VLAN 43 = Cucuron Caméras.

2.3 Le VLAN natif et le VLAN de gestion

Deux VLANs ont un rôle particulier qu'il faut bien comprendre :

📌 VLAN natif

Le VLAN natif est le VLAN utilisé sur un port trunk pour les trames non étiquetées (trames sans tag 802.1Q). Par défaut sur Cisco, c'est le VLAN 1. Les trames de ce VLAN sont transmises sans étiquette sur le trunk, ce qui peut créer des vulnérabilités (VLAN hopping).

Bonne pratique : changer le VLAN natif vers un VLAN inutilisé (ex. VLAN 99) pour éviter que des trames non étiquetées se retrouvent accidentellement dans un VLAN sensible.

📌 VLAN de gestion

Le VLAN de gestion (ou VLAN d'administration) est le VLAN sur lequel on attribue une adresse IP au switch pour l'administrer à distance (via SSH ou Telnet). Dans le projet Provence Évasion, c'est le VLAN x4 de chaque site. Par exemple, le switch de Roussillon a l'IP 192.168.24.2 (VLAN 24, Administration).

Important : si le VLAN de gestion n'est pas correctement configuré, le switch n'est pas joignable depuis le réseau — ce qui rend toute administration à distance impossible.

3. Ports Access et ports Trunk

3.1 Le port Access — un port, un VLAN

Un port Access est un port de switch configuré pour appartenir à un seul VLAN. C'est le type de port qui relie les équipements terminaux (PC, imprimante, caméra IP, point d'accès WiFi…) au switch.

Quand une trame arrive sur un port Access, le switch connaît déjà son VLAN : c'est celui du port. La trame ne porte pas de tag VLAN — elle est non étiquetée. Le switch ajoute mentalement le tag de façon interne pour router la trame dans le bon VLAN.

Schéma : port Access
PC-Personnel ────[Fa0/1 VLAN 21 Access]──── SWITCH PC-WiFi ────[Fa0/2 VLAN 22 Access]──── SWITCH Caméra IP ────[Fa0/3 VLAN 23 Access]──── SWITCH → Chaque port connaît son VLAN, la trame reste non étiquetée côté équipement.
Configuration d'un port Access — Cisco IOS Switch# configure terminal Switch(config)# interface FastEthernet0/1 Switch(config-if)# switchport mode access Switch(config-if)# switchport access vlan 21 ← affecte ce port au VLAN 21 Switch(config-if)# description PC-Personnel-Roussillon ← optionnel mais recommandé Switch(config-if)# end Switch# write memory ← sauvegarde la configuration
⚠ Erreur fréquente : Si vous configurez un port en mode Access sur un VLAN qui n'a pas encore été créé sur le switch, Cisco IOS affiche un message d'avertissement. Le port sera en mode suspendu jusqu'à la création du VLAN. Toujours créer les VLANs avant d'affecter les ports.

3.2 Le port Trunk — un câble, tous les VLANs

Un port Trunk est un port configuré pour transporter plusieurs VLANs simultanément sur un seul câble physique. Il est utilisé pour relier :

  • Deux switches entre eux (interconnexion de niveau 2)
  • Un switch et un routeur (pour le routage inter-VLAN)
  • Un switch et un point d'accès WiFi géré (qui gère plusieurs SSIDs = plusieurs VLANs)

Pour identifier à quel VLAN appartient chaque trame sur un trunk, on utilise le protocole 802.1Q (prononcé "dot 1 Q"). Ce protocole ajoute un tag de 4 octets dans l'en-tête de chaque trame Ethernet, contenant notamment le numéro de VLAN (12 bits = 4094 VLANs possibles).

Schéma : port Trunk entre switch et routeur
SWITCH ROUTEUR [Gi0/1 TRUNK]───────────────────[Gi0/0] ↑ Transporte VLAN 21 + 22 + 23 + 24 Chaque trame est étiquetée 802.1Q avec son VID
Configuration d'un port Trunk — Cisco IOS Switch# configure terminal Switch(config)# interface GigabitEthernet0/1 Switch(config-if)# switchport mode trunk Switch(config-if)# switchport trunk allowed vlan 21,22,23,24 ← VLANs autorisés Switch(config-if)# switchport trunk native vlan 99 ← VLAN natif ≠ VLAN 1 Switch(config-if)# description TRUNK-vers-Routeur Switch(config-if)# end Switch# write memory
CritèrePort AccessPort Trunk
Nombre de VLANs 1 seul VLAN Plusieurs VLANs simultanément
Trafic Trames non étiquetées (sans tag 802.1Q) Trames étiquetées 802.1Q (sauf VLAN natif)
Équipement connecté PC, imprimante, caméra, téléphone IP… Autre switch, routeur, AP WiFi multi-SSID…
Commande Cisco switchport mode access switchport mode trunk
VLAN natif N/A (port mono-VLAN) VLAN des trames non étiquetées (défaut : VLAN 1)

4. Création des VLANs sur un switch Cisco

4.1 Procédure complète de création d'un VLAN

Avant d'affecter un VLAN à des ports, il faut le créer dans la base de données VLAN du switch. Cette opération se fait en mode de configuration globale.

Création des VLANs — Site de Roussillon (exemple complet) Switch# configure terminal ─── Création des 4 VLANs du site de Roussillon ─── Switch(config)# vlan 21 Switch(config-vlan)# name Personnel-Roussillon Switch(config-vlan)# exit Switch(config)# vlan 22 Switch(config-vlan)# name Clients-WiFi-Roussillon Switch(config-vlan)# exit Switch(config)# vlan 23 Switch(config-vlan)# name Cameras-IP-Roussillon Switch(config-vlan)# exit Switch(config)# vlan 24 Switch(config-vlan)# name Administration-Roussillon Switch(config-vlan)# exit Switch(config)# end Switch# show vlan brief ← vérifie les VLANs créés Switch# write memory

4.2 Configurer l'interface de gestion (VLAN d'administration)

Pour administrer un switch à distance (via SSH par exemple), il faut lui attribuer une adresse IP. Sur un switch de niveau 2 (couche 2), cette adresse est configurée sur une interface VLAN virtuelle, appelée SVI (Switched Virtual Interface).

Configuration de l'interface de gestion — VLAN 24 (Administration Roussillon) Switch(config)# interface vlan 24 Switch(config-if)# ip address 192.168.24.2 255.255.255.240 ← IP de gestion du switch Switch(config-if)# no shutdown ← active l'interface Switch(config-if)# exit Switch(config)# ip default-gateway 192.168.24.1 ← passerelle du VLAN admin Switch(config)# end Switch# show interfaces vlan 24 ← vérifie l'interface
⚠ Important : L'interface VLAN d'administration ne sera active (up/up) que si au moins un port physique du switch est configuré dans ce VLAN ET qu'un équipement est connecté à ce port. Sans port actif dans le VLAN 24, l'interface de gestion reste down.

5. Le routage inter-VLAN — Router-on-a-Stick

5.1 Pourquoi les VLANs ne communiquent pas entre eux

Par définition, les VLANs sont des réseaux logiquement isolés. Un PC du VLAN 21 (Personnel) ne peut pas communiquer directement avec un PC du VLAN 22 (Clients WiFi), même s'ils sont branchés sur le même switch physique. C'est précisément le but de la segmentation.

Cependant, certaines communications entre VLANs sont légitimes et nécessaires. Par exemple :

  • Le personnel doit pouvoir accéder aux serveurs du siège (VLAN 11 → VLAN 15)
  • L'enregistreur NVR du siège doit pouvoir récupérer les flux des caméras des 4 sites (VLAN 15 → VLAN x3)
  • Tous les VLANs doivent pouvoir accéder à Internet (via la passerelle WAN)

Pour permettre ces échanges contrôlés, il faut un équipement de niveau 3 : un routeur.

5.2 Le principe Router-on-a-Stick (RoaS)

Le Router-on-a-Stick est une technique qui consiste à connecter un routeur au switch via un seul câble physique (trunk 802.1Q), et à créer sur le routeur des sous-interfaces logiques, une par VLAN. Chaque sous-interface porte l'adresse IP de la passerelle du VLAN correspondant.

Schéma Router-on-a-Stick — Site de Roussillon
┌──────────────────────────────────────┐ │ ROUTEUR 1941 │ │ Gi0/0.21 → 192.168.21.1/26 │ │ Gi0/0.22 → 192.168.22.1/25 │ │ Gi0/0.23 → 192.168.23.1/28 │ │ Gi0/0.24 → 192.168.24.1/28 │ └──────────────┬───────────────────────┘ │ Trunk 802.1Q (Gi0/1 switch ↔ Gi0/0 routeur) │ VLAN 21 + 22 + 23 + 24 ┌──────────────┴───────────────────────┐ │ SWITCH 2960 │ │ Fa0/1 → Access VLAN 21 (Personnel)│ │ Fa0/2 → Access VLAN 22 (WiFi) │ │ Fa0/3 → Access VLAN 23 (Caméras) │ │ Fa0/4 → Access VLAN 24 (Admin) │ │ Gi0/1 → Trunk vers Routeur │ └──────────────────────────────────────┘

5.3 Configuration du routeur — sous-interfaces 802.1Q

Sur le routeur, chaque VLAN est représenté par une sous-interface de la forme GigabitEthernet0/0.VID. On encapsule la sous-interface en 802.1Q avec le bon VID, puis on lui attribue l'adresse IP qui sera la passerelle de ce VLAN.

Configuration Router-on-a-Stick — Routeur Roussillon (Cisco 1941) Router# configure terminal ─── Interface physique : on l'active sans IP ─── Router(config)# interface GigabitEthernet0/0 Router(config-if)# no shutdown Router(config-if)# exit ─── Sous-interface VLAN 21 — Personnel ─── Router(config)# interface GigabitEthernet0/0.21 Router(config-subif)# encapsulation dot1Q 21 ← tag 802.1Q = VLAN 21 Router(config-subif)# ip address 192.168.21.1 255.255.255.192 ← passerelle VLAN 21 Router(config-subif)# exit ─── Sous-interface VLAN 22 — Clients WiFi ─── Router(config)# interface GigabitEthernet0/0.22 Router(config-subif)# encapsulation dot1Q 22 Router(config-subif)# ip address 192.168.22.1 255.255.255.128 Router(config-subif)# exit ─── Sous-interface VLAN 23 — Caméras IP ─── Router(config)# interface GigabitEthernet0/0.23 Router(config-subif)# encapsulation dot1Q 23 Router(config-subif)# ip address 192.168.23.1 255.255.255.240 Router(config-subif)# exit ─── Sous-interface VLAN 24 — Administration ─── Router(config)# interface GigabitEthernet0/0.24 Router(config-subif)# encapsulation dot1Q 24 Router(config-subif)# ip address 192.168.24.1 255.255.255.240 Router(config-subif)# exit Router(config)# end Router# show ip interface brief ← vérifie toutes les sous-interfaces Router# write memory

5.4 Vérification et test du routage inter-VLAN

Pour vérifier que le routage inter-VLAN fonctionne, on utilise les commandes classiques de diagnostic réseau :

Commandes de vérification — Routeur et Switch Sur le routeur : Router# show ip interface brief ← état de toutes les interfaces Router# show ip route ← table de routage (routes Connected) Router# ping 192.168.22.10 ← test de connectivité vers un hôte Sur le switch : Switch# show vlan brief ← liste des VLANs et leurs ports Switch# show interfaces trunk ← état des ports trunk et VLANs autorisés Switch# show interfaces FastEthernet0/1 switchport ← détail d'un port
⚠ Piège courant : Si show ip route ne montre pas les routes "Connected" pour vos VLANs, vérifiez que les sous-interfaces sont bien en état up/up (show ip interface brief). Une sous-interface reste down si l'interface physique parente est éteinte (shutdown) ou si le câble trunk n'est pas actif.

🎯 Synthèse de la séance

Un switch ne dispose d'aucune interface de configuration, tandis qu'un switch permet de créer des VLANs et de configurer chaque port.

Un VLAN crée un domaine de broadcast . Les trames broadcast restent confinées à leur propre .

Un port appartient à un seul VLAN et transporte des trames . Un port transporte plusieurs VLANs grâce au protocole .

La technique permet de router entre plusieurs VLANs avec un seul câble physique entre le switch et le routeur, en utilisant des sur le routeur.

Sur un routeur Cisco, la commande encapsulation dot1Q associe une sous-interface à un VLAN précis.
