1. Rappel : la table de routage et son fonctionnement

📍 Mise en situation : Lors de la séance S5, vous avez construit la topologie complète des 4 sites de Provence Évasion et commencé à configurer le routage statique entre Apt et Roussillon. Cucuron reste à interconnecter. Avant d'aller plus loin, Laurent Ferracci (PDG d'Axentis, votre cabinet) signale que certains pings inter-sites échouent. Objectif de la séance : savoir lire, interpréter et déboguer une table de routage pour identifier l'origine des pannes.

1.1 À quoi sert la table de routage ?

Un routeur doit savoir où envoyer chaque paquet IP qu'il reçoit. Pour cela, il consulte sa table de routage (aussi appelée routing table ou RIB — Routing Information Base). C'est une liste ordonnée de règles : pour chaque réseau de destination possible, quelle est la prochaine étape (next-hop) ou quelle interface de sortie utiliser ?

📌 Règle fondamentale : longest prefix match

Quand un paquet arrive, le routeur compare l'adresse IP de destination avec toutes les entrées de sa table de routage. Il choisit la route avec le masque le plus long (le plus précis). Si aucune route ne correspond, il utilise la route par défaut (0.0.0.0/0) si elle existe — sinon, le paquet est abandonné (dropped) et le routeur génère un message ICMP "Destination Unreachable".

1.2 Structure d'une entrée de table de routage Cisco

Voici un extrait typique de show ip route sur le routeur du siège d'Apt :

Routeur-Apt# show ip route Codes: C - connected, S - static, R - RIP, M - mobile, B - BGP D - EIGRP, EX - EIGRP external, O - OSPF, IA - OSPF inter area * - candidate default, U - per-user static route C 192.168.10.0/24 is directly connected, GigabitEthernet0/0.11 ← réseau local directement connecté C 192.168.11.0/28 is directly connected, GigabitEthernet0/0.13 C 192.168.12.0/28 is directly connected, GigabitEthernet0/0.14 C 192.168.13.0/29 is directly connected, GigabitEthernet0/0.15 C 10.0.0.0/30 is directly connected, Serial0/0/0 ← liaison WAN Apt—Roussillon S 192.168.20.0/22 [1/0] via 10.0.0.2 ← route statique vers Roussillon S 192.168.30.0/22 [1/0] via 10.0.1.2 ← route statique vers Saignon S* 0.0.0.0/0 [1/0] via 203.0.113.1 ← route par défaut (vers Internet)

2. Les commandes de diagnostic essentielles

2.1 show ip route — lire la table de routage complète

C'est la commande de base. Elle affiche l'intégralité des routes connues du routeur, avec leur type, leur next-hop et leur interface de sortie. On peut la filtrer :

Variantes de show ip route Routeur-Apt# show ip route ← affiche toute la table Routeur-Apt# show ip route 192.168.40.0 ← cherche une route vers ce réseau Routeur-Apt# show ip route static ← affiche uniquement les routes statiques Routeur-Apt# show ip route connected ← affiche uniquement les réseaux directement connectés

2.2 show interfaces — état des interfaces

Une route ne peut être active que si l'interface correspondante est UP/UP. La commande show interfaces (ou show ip interface brief) permet de vérifier l'état physique et logique de chaque interface.

Routeur-Apt# show ip interface brief Interface IP-Address OK? Method Status Protocol GigabitEthernet0/0 unassigned YES unset up up GigabitEthernet0/0.11 192.168.10.1 YES manual up up GigabitEthernet0/0.13 192.168.11.1 YES manual up up Serial0/0/0 10.0.0.1 YES manual up up Serial0/0/1 10.0.1.1 YES manual administratively down down ⚠ PROBLÈME !
⚠ À retenir : Si une interface affiche administratively down, c'est qu'elle a été désactivée manuellement (shutdown). Il faut utiliser no shutdown pour la réactiver. Si l'interface est down/down sans "administratively", c'est un problème physique (câble, SFP, autre équipement).

2.3 ping et traceroute — tester la connectivité

Ces deux commandes sont complémentaires. Le ping confirme qu'une destination est joignable. Le traceroute (ou tracert sous Windows) révèle le chemin emprunté saut par saut.

Utilisation de ping étendu (depuis le routeur Cisco) Routeur-Apt# ping 192.168.41.10 Type escape sequence to abort. Sending 5, 100-byte ICMP Echos to 192.168.41.10, timeout is 2 seconds: ..... Success rate is 0 percent (0/5) ← 0% = la destination est injoignable Routeur-Apt# ping 192.168.21.10 Type escape sequence to abort. Sending 5, 100-byte ICMP Echos to 192.168.21.10, timeout is 2 seconds: !!!!! Success rate is 100 percent (5/5) ← 5/5 = connectivité OK
Utilisation de traceroute Routeur-Apt# traceroute 192.168.31.10 Type escape sequence to abort. Tracing the route to 192.168.31.10 1 10.0.1.2 8 msec 4 msec 4 msec 2 192.168.31.10 12 msec 8 msec 8 msec ← 2 sauts, route valide ✓

3. La passerelle par défaut — côté équipement terminal

3.1 Pourquoi configurer une passerelle par défaut ?

Un PC ou un serveur ne possède pas de table de routage aussi élaborée qu'un routeur. Il ne connaît que son propre réseau local. Pour joindre un autre réseau (un autre site, Internet…), il doit envoyer le paquet à une machine de confiance chargée de le router : sa passerelle par défaut (default gateway). C'est l'adresse IP du routeur (ou de la sous-interface Router-on-a-Stick) du VLAN concerné.

Schéma : rôle de la passerelle par défaut
PC-Personnel (192.168.21.10) ──→ Switch Roussillon ──→ Routeur (192.168.21.1) │ Paquet destiné à 192.168.41.5 (Cucuron) │ consulte sa table de routage → Le PC ne connaît pas ce réseau ↓ → Il envoie le paquet à sa GW : 192.168.21.1 Routeur route vers 10.0.2.2 (Cucuron)

3.2 Vérifier et configurer la passerelle par défaut

Si la passerelle par défaut est absente ou incorrecte, le PC peut pinguer son réseau local mais pas les autres sites. C'est une cause fréquente de panne inter-sites.

✅ Configuration correcte

PC-Personnel-1 (Roussillon)
IP : 192.168.21.10 / 255.255.255.0
Passerelle : 192.168.21.1 ← sous-interface routeur

→ Peut joindre tous les autres sites.

❌ Erreur courante

PC-Personnel-1 (Roussillon)
IP : 192.168.21.10 / 255.255.255.0
Passerelle : 192.168.21.10 ← adresse du PC lui-même !

→ Ne peut pas quitter son réseau local.

4. Méthodologie de troubleshooting réseau

4.1 Approche systématique : du bas vers le haut

Face à une panne réseau, il ne faut pas "tester au hasard". La méthode professionnelle consiste à remonter le modèle OSI couche par couche, ou à utiliser la méthode de bissection (tester le milieu du chemin pour diviser le problème).

🔧 Méthode par couche — troubleshooting inter-sites
1
Couche 1 (Physique) — Le câble est-il branché ? L'interface est-elle UP ? → show interfaces
2
Couche 2 (Liaison) — Les VLANs sont-ils bien configurés ? Le port trunk passe-t-il les bons VLANs ? → show vlan brief, show interfaces trunk
3
Couche 3 — Local — Le PC a-t-il la bonne IP et la bonne passerelle ? Peut-il pinguer sa propre passerelle ? → ipconfig (Windows) ou ip a (Linux)
4
Couche 3 — Routeur source — Le routeur du site source a-t-il une route vers le réseau de destination ? → show ip route
5
Couche 3 — Routeur destination — Le routeur du site destination a-t-il une route de retour vers le réseau source ? → show ip route sur le routeur distant
6
Traçage du chemin complet — Utiliser traceroute pour visualiser les sauts et identifier où la chaîne se coupe.
⚠ Oubli classique — la route de retour ! Une erreur très fréquente : on configure la route aller (Apt → Cucuron) mais on oublie la route retour (Cucuron → Apt). Le ping part mais ne revient pas → 0% de succès. Toujours vérifier les deux sens.

5. La route par défaut — ouvrir vers l'extérieur

5.1 Pourquoi une route par défaut ?

Les routeurs de Provence Évasion connaissent les réseaux des 4 sites grâce aux routes statiques. Mais qu'en est-il d'Internet ? Il est impossible de mémoriser les centaines de milliers de réseaux Internet dans une table de routage manuelle. La solution : la route par défaut, qui dit au routeur "pour tout réseau que tu ne connais pas, envoie le paquet par ici".

Configuration d'une route par défaut sur le routeur d'Apt Routeur-Apt(config)# ip route 0.0.0.0 0.0.0.0 203.0.113.1 ← vers la passerelle Internet (FAI) Vérification : Routeur-Apt# show ip route S* 0.0.0.0/0 [1/0] via 203.0.113.1 ← le * indique que c'est la candidate par défaut

5.2 Propagation de la route par défaut vers les sites secondaires

Les routeurs des sites secondaires (Roussillon, Saignon, Cucuron) doivent eux aussi savoir comment atteindre Internet. La solution la plus simple : configurer une route par défaut qui pointe vers le routeur d'Apt, qui se charge ensuite de transmettre vers le FAI.

Route par défaut sur les sites secondaires (exemple : Roussillon) Routeur-Roussillon(config)# ip route 0.0.0.0 0.0.0.0 10.0.0.1 ← next-hop = routeur d'Apt côté WAN
