1. Contexte et objectifs de l'atelier

🏨 Mission Axentis : Le site d'Apt (siège de Provence Évasion) compte 3 VLANs actifs nécessitant une distribution automatique d'adresses IP : VLAN 11 (Administration), VLAN 12 (Accueil) et VLAN 13 (Clients WiFi). Votre mission est de configurer le service DHCP directement sur le routeur Cisco du site, à partir du fichier Packet Tracer fourni par votre professeur.

À l'issue de cet atelier, vous serez capable de :

  • Configurer des exclusions DHCP sur un routeur Cisco IOS
  • Créer des pools DHCP par VLAN avec les bons paramètres
  • Vérifier l'attribution automatique des adresses sur les postes clients
  • Créer une réservation DHCP liée à une adresse MAC
  • Utiliser les commandes de vérification DHCP
📋 Fichier de départ : Ouvrez le fichier PE_Apt_S3_Atelier2.pkt (topology d'Apt avec VLANs et trunk déjà configurés). Sauvegardez-le immédiatement sous PE_Apt_S4_Atelier1.pkt avant toute modification.

2. Préparation — Passage des PCs en mode DHCP

Avant de configurer le serveur DHCP sur le routeur, il faut d'abord passer tous les PC clients en mode DHCP dans Packet Tracer, pour qu'ils demandent automatiquement leur configuration réseau.

Étape 1 Cliquez sur PC-Admin-1 → onglet Desktop → IP Configuration.
Étape 2 Sélectionnez l'option DHCP (au lieu de Static). Le PC affiche immédiatement "DHCP request failed" — c'est normal, le serveur n'est pas encore configuré.
Étape 3 Répétez pour PC-Admin-2, PC-Accueil-1, PC-Accueil-2 et PC-WiFi-1.
Étape 4 L'imprimante réseau reste en IP statique (192.168.12.5 — elle aura une réservation).
