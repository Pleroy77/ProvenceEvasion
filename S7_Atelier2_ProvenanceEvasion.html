1. Enregistrements CNAME — les alias DNS

1.1 Principe du CNAME

Un enregistrement CNAME (Canonical Name) crée un alias : il fait pointer un nom vers un autre nom (et non vers une IP directement). Cela permet d'avoir plusieurs noms pour un même équipement, sans dupliquer les enregistrements A.

Exemple : fichiers.provence-evasion.local → nas.provence-evasion.local → 192.168.15.2. Si l'IP du NAS change, on ne modifie que l'enregistrement A de nas — le CNAME suit automatiquement.

⚠ Règle importante : Un CNAME ne peut jamais pointer vers une adresse IP directement — uniquement vers un autre nom de domaine. Et on ne peut pas créer un CNAME pour la racine d'une zone (le sommet de zone doit avoir un enregistrement A ou NS, pas un CNAME).

1.2 Création des CNAME dans Packet Tracer

🛠 Consigne : Ouvrez l'onglet Services → DNS du Serveur-DNS. Ajoutez les enregistrements CNAME suivants. Dans Packet Tracer, utilisez le type "CNAME" dans le menu déroulant et saisissez le nom canonique (cible) dans le champ Address.

2. Zone de résolution inverse — enregistrements PTR

2.1 Rappel et construction du nom de zone inverse

La zone inverse permet de trouver le nom associé à une adresse IP (opération inverse du DNS classique). Pour le réseau 192.168.15.0/24, la zone inverse se nomme 15.168.192.in-addr.arpa — les trois premiers octets du réseau sont inversés et suivis de .in-addr.arpa.

📌 Construction d'un enregistrement PTR

Pour créer l'enregistrement PTR de nas.provence-evasion.local (192.168.15.2) :

  • Name : 2.15.168.192.in-addr.arpa (l'IP complète inversée + .in-addr.arpa)
  • Type : PTR
  • Address : nas.provence-evasion.local (le FQDN cible)

Dans Packet Tracer, on utilise le même serveur DNS pour les enregistrements PTR — il n'y a pas de zone "inverse" séparée dans l'interface graphique, on ajoute simplement des enregistrements de type PTR.

3. Simulation du forwarder — résolution de noms externes

3.1 Mise en place du serveur DNS "Internet" simulé

Dans un environnement réel, quand le DNS interne ne connaît pas un nom (ex : www.meteo.fr), il transfère la requête vers un résolveur public. Dans Packet Tracer, on va simuler ce comportement en ajoutant un deuxième serveur DNS qui jouera le rôle d'un serveur DNS externe.

① Ajoutez un serveur Serveur-DNS-EXTERNE à l'adresse 8.8.8.1, relié directement au Routeur-APT (interface Fa1/0 ou une interface libre) via un câble direct
② Activez le service DNS sur ce serveur et créez un enregistrement A : www.internet-simule.com → 8.8.8.10
③ Sur le Routeur-APT, ajoutez une route vers le réseau 8.8.8.0/24 : ip route 8.8.8.0 255.255.255.0 [interface ou nexthop vers Srv-Ext]
④ Sur le Serveur-DNS interne, configurez le forwarder : dans l'onglet DNS, cherchez le champ Default Server (ou DNS Server) et saisissez 8.8.8.1
⚠ Limitation Packet Tracer : Selon la version, le forwarder peut ne pas être configurable graphiquement sur un serveur Packet Tracer basique. Si c'est le cas, notez-le et décrivez comment cela fonctionnerait en production (la question Q3.2 reste à traiter par analyse).

4. Scénario de diagnostic — panne DNS chez Provence Évasion

🚨 Situation de panne — Appel de Laurent Ferracci

Le lundi matin, Laurent Ferracci appelle Axentis : "Depuis ce matin, personne ne peut accéder au NAS depuis Roussillon. Quand on tape \\fichiers.provence-evasion.local dans l'explorateur Windows, on obtient 'Impossible de trouver le chemin réseau'. Par contre, en tapant \\192.168.15.2, ça fonctionne. Le problème vient d'hier soir : un technicien a 'fait quelque chose' sur le réseau."

Votre mission : diagnostiquer la panne et la corriger dans Packet Tracer.

4.1 Reproduction de la panne dans Packet Tracer

🛠 Consigne : Pour simuler la panne, effectuez l'une des modifications suivantes sur votre topologie (au choix, ou votre enseignant vous indique laquelle) :
  • Scénario A : Modifiez l'IP du serveur DNS configurée sur PC-ROUSSILLON (par ex. mettez 192.168.15.99 au lieu de 192.168.15.1)
  • Scénario B : Supprimez l'enregistrement CNAME fichiers.provence-evasion.local du service DNS
  • Scénario C : Désactivez le service DNS sur le serveur (DNS Service → OFF)
