1. Analyse — Politique de sécurité inter-sites

📍 Mise en situation : Les ACL du site de Roussillon (Atelier 1) sont opérationnelles. Mais Laurent Ferracci découvre un nouveau problème : « Un client WiFi de notre bastide de Cucuron a réussi à accéder à nos serveurs internes depuis son smartphone ! » La protection doit s'étendre aux 4 sites de manière cohérente. Vous devez déployer une ACL centralisée sur ROUTEUR-APT pour les flux inter-sites, et des ACL locales spécifiques sur Saignon (comptabilité sensible) et Cucuron.

3. ACL locales — Saignon et Cucuron

⏱ ~35 min — environ 18 min par site

3.1 ROUTEUR-SAIGNON — Protection de la comptabilité (COMPTA-PROTECTION)

Objectif : le Personnel de Saignon (VLAN 41) ne peut pas accéder en SSH aux équipements réseau. La Comptabilité (VLAN 42) peut joindre les serveurs Apt (SMB TCP 445 + DNS UDP 53). Tout autre flux depuis VLAN 42 est bloqué (pas d'internet direct, pas d'accès aux VLANs Personnel des autres sites).

📝 ACL COMPTA-PROTECTION — Saignon

3.2 ROUTEUR-CUCURON — Isolation WiFi locale (CUCURON-WIFI-ISO)

Par symétrie avec Roussillon, le WiFi de Cucuron (VLAN 64) ne doit pas accéder aux réseaux privés 172.16.0.0/16 ni aux liaisons WAN 10.0.0.0/8, mais peut accéder à Internet.

📝 ACL CUCURON-WIFI-ISO — Cucuron

5. Débogage avancé — 7 scénarios inter-sites

🎯 Objectif : Ces scénarios se concentrent sur des erreurs spécifiques aux ACL inter-sites et aux interactions entre plusieurs ACL actives. Pour chaque scénario : analyse sur papier → saisie en PT → correction → vérification.

5.1 D1 — La Comptabilité ne peut plus accéder aux serveurs (règle deny parasite)

🐛 Symptôme : le chef comptable de Saignon ne peut plus ouvrir les fichiers partagés
ip access-list extended INTER-SITES-SEC
 deny ip 172.16.24.0 0.0.0.255 172.16.10.0 0.0.0.255
 deny ip 172.16.44.0 0.0.0.255 172.16.10.0 0.0.0.255
 deny ip 172.16.64.0 0.0.0.255 172.16.10.0 0.0.0.255
 deny ip 172.16.43.0 0.0.0.255 any
 deny ip 172.16.63.0 0.0.0.255 any
 deny ip 172.16.42.0 0.0.0.255 any   ← ligne ajoutée par erreur
 permit tcp 172.16.42.0 0.0.0.255 172.16.10.0 0.0.0.255 eq 445
 permit ip any any
📝 Analyse D1

5.2 D2 — L'ACL n'est pas appliquée : le compteur reste à zéro

🐛 Symptôme : les clients WiFi de Saignon accèdent encore aux serveurs malgré une ACL correcte
Router# show access-lists INTER-SITES-SEC
Extended IP access list INTER-SITES-SEC
  10 deny ip 172.16.24.0 0.0.0.255 172.16.10.0 0.0.0.255
  20 deny ip 172.16.44.0 0.0.0.255 172.16.10.0 0.0.0.255  ← 0 matches malgré les tests
  30 permit ip any any (12 matches)

Router# show ip interface Serial0/0/1
  Inbound  access list is not set      ← clé du problème !
📝 Analyse D2

5.3 D3 — Interaction entre deux ACL : le flux légitime est bloqué deux fois

🐛 Symptôme : le Personnel de Saignon ne peut plus ping le Personnel de Roussillon (flux inter-sites légitime)
! Sur ROUTEUR-SAIGNON :
ip access-list extended COMPTA-PROTECTION
 deny tcp 172.16.41.0 0.0.0.255 any eq 22
 deny ip 172.16.41.0 0.0.0.255 any   ← trop large ! bloque tout le Personnel
 permit tcp 172.16.42.0 0.0.0.255 172.16.10.0 0.0.0.255 eq 445
 permit ip any any
📝 Analyse D3

5.4 D4 — Wildcard trop large : blocage d'un VLAN légitime

🐛 Symptôme : la Direction de Roussillon (172.16.22.x) ne peut plus accéder aux serveurs Apt
ip access-list extended INTER-SITES-SEC
 deny ip 172.16.20.0 0.0.7.255 172.16.10.0 0.0.0.255   ← wildcard suspect
 permit ip any any
📝 Analyse D4

5.5 D5 — ACL sur ROUTEUR-APT appliquée en OUT au lieu de IN

🐛 Symptôme : INTER-SITES-SEC existe mais les WiFi de tous les sites accèdent encore aux serveurs
! ACL INTER-SITES-SEC correctement écrite MAIS :
interface Serial0/0/0
 ip access-group INTER-SITES-SEC out    ← OUT au lieu de IN

! Résultat show ip interface Serial0/0/0 :
Inbound  access list is not set
Outbound access list is INTER-SITES-SEC
📝 Analyse D5

5.6 D6 — Suppression accidentelle d'une interface ACL (no ip access-group)

🐛 Symptôme : après une mise à jour du routeur, les WiFi de Cucuron accèdent à nouveau aux réseaux privés locaux
! show ip interface G0/0.64 :
Inbound  access list is not set    ← CUCURON-WIFI-ISO a disparu !
Outbound access list is not set

! L'ACL existe toujours dans la config :
Router# show access-lists CUCURON-WIFI-ISO
Extended IP access list CUCURON-WIFI-ISO
  10 deny ip 172.16.64.0 0.0.0.255 172.16.0.0 0.0.255.255
  20 deny ip 172.16.64.0 0.0.0.255 10.0.0.0 0.255.255.255
  30 permit ip 172.16.64.0 0.0.0.255 any
📝 Analyse D6

5.7 D7 — Conflit entre INTER-SITES-SEC et trafic retour : connexion TCP asymétrique

🐛 Symptôme : le SYN de Comptabilité Saignon → serveur TCP 445 passe, mais la session TCP ne s'établit jamais
! Observé en mode Simulation Packet Tracer :
! SYN (172.16.42.x → 172.16.10.50 TCP 445) : FORWARDED sur ROUTEUR-APT Serial0/0/1
! SYN-ACK retour (172.16.10.50 → 172.16.42.x) : DROPPED sur ROUTEUR-APT Serial0/0/1
📝 Analyse D7
