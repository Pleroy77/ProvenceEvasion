⏱ ~20 min

Objectif : le personnel (VLAN 21) peut utiliser SMB (TCP 445) et DNS (UDP 53) vers Apt, mais pas SSH (TCP 22) ni HTTP (TCP 80) vers les équipements réseau. Tout autre trafic est autorisé.

4. Débogage — 6 scénarios de configuration incorrecte

🎯 Objectif : Pour chaque scénario, analysez d'abord sur papier ce qui se passe, puis saisissez la configuration dans Packet Tracer, observez les symptômes, corrigez et vérifiez. Chaque scénario est indépendant et se fait sur une topologie simple (1 routeur, 2 VLANs).

4.1 Scénario D1 — Le permit any any en tête d'ACL

🐛 ACL-D1 — Configuration à analyser
ip access-list extended ACL-D1
 permit ip any any
 deny ip 172.16.24.0 0.0.0.255 172.16.10.0 0.0.0.255
 deny ip 172.16.23.0 0.0.0.255 any
interface GigabitEthernet0/0.24
 ip access-group ACL-D1 in
📝 Analyse D1

4.2 Scénario D2 — Wildcard incorrect (masque de sous-réseau inversé)

🐛 ACL-D2 — Wildcard erroné
ip access-list extended ACL-D2
 deny ip 172.16.24.0 255.255.255.0 172.16.10.0 0.0.0.255
 permit ip any any
📝 Analyse D2

4.3 Scénario D3 — ACL appliquée en OUT au lieu de IN

🐛 ACL-D3 — Sens d'application incorrect
ip access-list extended ACL-D3
 deny ip 172.16.24.0 0.0.0.255 172.16.10.0 0.0.0.255
 permit ip any any
interface GigabitEthernet0/0.24
 ip access-group ACL-D3 out    ← OUT au lieu de IN
📝 Analyse D3

4.4 Scénario D4 — Règle deny trop large qui bloque les flux légitimes

🐛 ACL-D4 — Blocage collatéral inattendu
ip access-list extended ACL-D4
 deny ip 172.16.23.0 0.0.0.255 172.16.10.0 0.0.0.255
 permit tcp 172.16.23.0 0.0.0.255 host 172.16.10.20 eq 8080
 deny ip 172.16.23.0 0.0.0.255 any
interface GigabitEthernet0/0.23
 ip access-group ACL-D4 in
📝 Analyse D4

4.5 Scénario D5 — Oubli du permit final : deny all implicite bloque tout

🐛 ACL-D5 — La règle implicite qui coupe le réseau
ip access-list extended ACL-D5
 deny tcp 172.16.21.0 0.0.0.255 any eq 22
 deny tcp 172.16.21.0 0.0.0.255 any eq 80
 permit tcp 172.16.21.0 0.0.0.255 172.16.10.0 0.0.0.255 eq 445
 permit udp 172.16.21.0 0.0.0.255 host 172.16.10.10 eq 53
 ! ← aucun permit général ici
interface GigabitEthernet0/0.21
 ip access-group ACL-D5 in
📝 Analyse D5

4.6 Scénario D6 — ACL sur la mauvaise sous-interface

🐛 ACL-D6 — Bonne ACL, mauvaise interface
ip access-list extended WIFI-RESTRICTION
 deny ip 172.16.24.0 0.0.0.255 172.16.10.0 0.0.0.255
 permit ip any any

interface GigabitEthernet0/0.21   ← VLAN 21 Personnel, pas VLAN 24 WiFi !
 ip access-group WIFI-RESTRICTION in
📝 Analyse D6
