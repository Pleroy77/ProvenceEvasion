1. Mise en situation — Provence Évasion et ses données partagées

🏨 Contexte : Le groupe Provence Évasion dispose désormais d'un réseau opérationnel sur ses 4 sites (S1–S7). Laurent Ferracci souhaite maintenant que ses équipes puissent partager des fichiers de manière sécurisée : les plannings de réservation, les fiches clients, les documents RH et les bilans comptables doivent être accessibles depuis tous les sites, mais pas par tout le monde. Un technicien d'Axentis Informatique va configurer un serveur de fichiers centralisé au siège d'Apt.

1.1 Qu'est-ce qu'un serveur de fichiers ?

Un serveur de fichiers (ou file server) est un équipement dédié au stockage et au partage de fichiers sur un réseau. Il centralise les données pour que plusieurs utilisateurs puissent y accéder simultanément, depuis n'importe quel poste du réseau.

📌 À retenir — Serveur de fichiers

Un serveur de fichiers offre plusieurs avantages :

  • Centralisation : toutes les données sont stockées à un seul endroit, facilitant les sauvegardes
  • Partage : plusieurs utilisateurs accèdent aux mêmes fichiers simultanément
  • Contrôle des accès : on peut définir qui peut lire, modifier ou supprimer chaque dossier
  • Disponibilité : accessible depuis tous les sites via le réseau WAN

1.2 Les protocoles de partage de fichiers

SMB / CIFS

Server Message Block — protocole Windows. Utilisé par défaut dans les environnements Microsoft. Les partages apparaissent sous la forme \\serveur\partage. C'est le protocole natif de Packet Tracer.

NFS

Network File System — protocole Linux/Unix. Plus utilisé dans les environnements mixtes ou sous Linux. Les dossiers partagés sont montés comme des disques locaux.

Dans le projet Provence Évasion (et dans Packet Tracer), nous utilisons le service FTP/TFTP fourni par les serveurs Cisco pour simuler le serveur de fichiers.

3. Lien avec les VLANs et la sécurité

3.1 Le rôle des VLANs dans la sécurité des partages

La segmentation en VLANs réalisée lors des séances précédentes joue un rôle essentiel dans la sécurisation du serveur de fichiers. Le contrôle des accès repose sur deux niveaux complémentaires :

Niveau réseau (VLANs + routage)
  • Le routeur décide quels VLANs peuvent atteindre le VLAN 15 (Serveurs)
  • Les clients WiFi (VLAN x2) sont bloqués au niveau du routeur
  • Les caméras IP (VLAN x3) n'ont pas besoin d'accéder aux fichiers
Niveau applicatif (droits sur partages)
  • Le serveur NAS gère ses propres droits par dossier
  • Un utilisateur du VLAN Personnel peut accéder à Réservations mais pas à RH
  • Les droits s'appliquent après que le réseau a autorisé la connexion
📌 À retenir — Défense en profondeur

La sécurité des données repose sur plusieurs couches :

  1. Couche réseau : les VLANs isolent les flux — un PC client WiFi ne peut pas physiquement atteindre le serveur
  2. Couche routage : le routeur filtre les communications inter-VLANs (préparation des ACL en S9)
  3. Couche applicative : le serveur NAS gère lui-même les droits d'accès par dossier

C'est le principe de la défense en profondeur : même si un attaquant franchit une couche, les suivantes le bloquent.

3.2 Adressage du serveur NAS dans la topologie Provence Évasion

Le serveur NAS est placé dans le VLAN 15 (Serveurs) au siège d'Apt. Il utilise une adresse statique conforme à la convention établie en S2 :

4. Configuration du serveur de fichiers dans Packet Tracer

4.1 Le service serveur dans Packet Tracer

Dans Packet Tracer, un serveur générique peut activer plusieurs services : DHCP, DNS, HTTP, FTP… Pour simuler un serveur de fichiers, nous utilisons le service FTP qui permet de transférer des fichiers entre un client et le serveur.

Vérification de la connectivité depuis un PC vers le NAS ─── Depuis PC-Personnel-Roussillon (192.168.21.10) ─── PC> ping 192.168.15.4 ← test de connectivité vers le NAS Reply from 192.168.15.4: bytes=32 time=2ms TTL=126 PC> ftp 192.168.15.4 ← connexion FTP au serveur de fichiers Trying to connect... 192.168.15.4 Connected to 192.168.15.4 220- Welcome to PT Ftp server Username: employe 331- Username ok, need password Password: 230- Logged in (passive mode On) ftp>

4.3 Gestion des utilisateurs et mots de passe

Dans Packet Tracer, le service FTP d'un serveur permet de créer des utilisateurs avec des droits distincts. On accède à la configuration via l'onglet Services → FTP du serveur :

  • Username : identifiant de l'utilisateur (ex : employe, direction)
  • Password : mot de passe de connexion
  • Permissions : Write, Read, Delete, Rename, List (cocher selon les droits)
⚠ Important : Dans Packet Tracer, tous les utilisateurs FTP ont accès aux mêmes fichiers. La segmentation réelle des partages par VLAN se fait via les ACL sur les routeurs (séance S9). Pour l'atelier S8, nous simulons les droits en créant des comptes distincts avec des permissions différentes.
