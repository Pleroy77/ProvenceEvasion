1. Pourquoi le routage ?

1.1 La limite des switches

Un switch opère au niveau 2 (couche liaison) : il transfère les trames Ethernet en se basant sur les adresses MAC. Il est donc capable de faire communiquer des équipements appartenant au même réseau logique (même sous-réseau IP).

Mais dès que deux équipements se trouvent sur des sous-réseaux différents — même s'ils sont reliés physiquement — le switch ne peut pas acheminer le trafic. Il faut un équipement de niveau 3 : le routeur.

Rappel : Dans la S3 (VLANs), vous avez déjà utilisé du routage inter-VLAN via Router-on-a-Stick pour faire communiquer des VLANs différents sur un même site. Ici, nous allons étendre ce principe entre des sites géographiquement distants.

1.2 Le rôle du routeur

Le routeur opère au niveau 3 (couche réseau). Il analyse l'adresse IP de destination de chaque paquet et décide vers quelle interface (et donc quel réseau) le transmettre. Pour cela, il consulte sa table de routage.

2. La table de routage

2.1 Principe général

La table de routage est une base de données interne au routeur. Elle associe un réseau de destination à un prochain saut (next-hop) ou à une interface de sortie. À la réception de chaque paquet, le routeur cherche l'entrée la plus précise correspondant à l'adresse IP de destination.

Règle du chemin le plus spécifique (longest prefix match) :
Si plusieurs entrées correspondent à la destination, le routeur choisit celle avec le masque le plus long (le préfixe /30 est préféré à /24, qui est préféré à /0).

3. La topologie WAN de Provence Évasion

3.1 Architecture étoile centrée sur Apt

Le siège social d'Apt est le point central de l'infrastructure. Les trois sites secondaires (Roussillon, Saignon, Cucuron) se connectent chacun directement à Apt via une liaison WAN point-à-point. Il n'y a pas de liaison directe entre les sites secondaires.

3.2 Plan d'adressage WAN

La plage WAN retenue par Axentis est 10.0.0.0/24, découpée en sous-réseaux /30 (4 adresses, 2 utilisables) pour chaque liaison point-à-point.

4. Configuration des routes statiques

4.1 Syntaxe Cisco IOS

La commande de configuration d'une route statique est :

Router(config)# ip route <réseau_dest> <masque> <next-hop_ou_interface>

Exemple : pour que Router-Apt atteigne le VLAN 21 de Roussillon (192.168.21.0/26) en passant par 10.0.0.2 :

Router-Apt(config)# ip route 192.168.21.0 255.255.255.192 10.0.0.2

Router-Apt doit connaître tous les réseaux de tous les sites secondaires. Pour Roussillon (4 VLANs), Saignon (4 VLANs) et Cucuron (4 VLANs), cela fait 12 routes statiques à saisir.

4.3 Route par défaut sur les sites secondaires

Au lieu de saisir toutes les routes vers Apt ET les autres sites secondaires, les routeurs secondaires utilisent une route par défaut (default route) : tout trafic sans correspondance plus précise est envoyé vers Router-Apt, qui se chargera du routage final.

! Sur Router-Roussillon : Router-Rous(config)# ip route 0.0.0.0 0.0.0.0 10.0.0.1 ! Sur Router-Saignon : Router-Saig(config)# ip route 0.0.0.0 0.0.0.0 10.0.0.5 ! Sur Router-Cucuron : Router-Cuc(config)# ip route 0.0.0.0 0.0.0.0 10.0.0.9
